Penjelasan Project

Sistem Pendukung Keputusan untuk Kredit Usaha Mikro

Dalam dunia perbankan dan lembaga keuangan mikro, proses penilaian kelayakan kredit merupakan tantangan yang kompleks. Setiap hari, petugas kredit harus mengevaluasi puluhan pengajuan dengan berbagai latar belakang finansial yang berbeda. Proses manual ini rentan terhadap bias subjektif dan inkonsistensi penilaian, yang dapat berakibat pada kerugian finansial atau hilangnya kesempatan bisnis.

Problem Statement

Tantangan utama dalam penilaian kredit mikro secara manual

  • Proses evaluasi memakan waktu berjam-jam per pengajuan
  • Keputusan bergantung pada subjektivitas petugas individual
  • Inkonsistensi penilaian antar petugas dan waktu
  • Kesalahan penilaian berakibat kredit macet atau opportunity loss
  • Tidak ada dokumentasi reasoning yang sistematis

Solusi Sistem

Pendekatan Machine Learning untuk keputusan kredit yang objektif

  • Algoritma Naive Bayes untuk klasifikasi probabilistik
  • 5 Fitur Kategorik: Riwayat Kredit, Lama Usaha, Pendapatan, Jaminan, Jumlah Pinjaman
  • Output Binary: Terima atau Tolak dengan probabilitas
  • Keputusan berbasis pola data historis yang tervalidasi
  • Transparansi penuh dengan confidence score

Manfaat Implementasi

Dampak nyata bagi operasional lembaga keuangan mikro

  • Kecepatan: Dari berjam-jam menjadi <100ms per prediksi
  • Objektif: Keputusan berbasis data, bukan intuisi
  • Konsisten: Input sama = output sama setiap waktu
  • Akurat: Divalidasi dengan 5-Fold Cross-Validation
  • Transparan: Probabilitas eksplisit untuk audit

Bagaimana Naive Bayes Bekerja

Algoritma, formula matematika, dan implementasi teknis dalam project ini

Naive Bayes adalah algoritma klasifikasi probabilistik berdasarkan Teorema Bayes dari abad ke-18. Algoritma menghitung probabilitas data masuk ke kelas tertentu berdasarkan fitur-fiturnya, dengan asumsi independensi antar fitur. Meski asumsi ini tidak selalu benar di dunia nyata, Naive Bayes tetap powerful untuk klasifikasi data kategorik.

Teorema Bayes
$$P(\text{Class}|\text{Features}) = \frac{P(\text{Features}|\text{Class}) \times P(\text{Class})}{P(\text{Features})}$$
Posterior Probability = (Likelihood × Prior) / Evidence
Probabilitas kelas given fitur = (Probabilitas fitur given kelas × Probabilitas kelas) / Probabilitas fitur
Naive Bayes Classifier
$$P(y|x_1, x_2, ..., x_n) = \frac{P(y) \prod_{i=1}^{n} P(x_i|y)}{P(x_1, x_2, ..., x_n)}$$
Di mana \(y\) adalah kelas (Terima/Tolak) dan \(x_1, x_2, ..., x_n\) adalah fitur input
Asumsi independensi: \(P(x_1, x_2, ..., x_n|y) = P(x_1|y) \times P(x_2|y) \times ... \times P(x_n|y)\)

Implementasi Teknis

Categorical Naive Bayes

Varian khusus Scikit-learn untuk data kategorik diskrit

  • Menggunakan CategoricalNB dari Scikit-learn
  • Optimal untuk fitur kategorik (Buruk/Cukup/Baik, Kecil/Sedang/Besar)
  • Distribusi multinomial untuk probabilitas kategori
  • Tidak ada asumsi distribusi normal seperti GaussianNB
  • Perfect fit untuk 5 fitur kategorik sistem kredit

Preprocessing Pipeline

Transformasi data sebelum masuk ke classifier

  • OrdinalEncoder: Konversi kategori → angka ordinal
  • Contoh mapping: Buruk=0, Cukup=1, Baik=2
  • Encoding konsisten untuk training & prediksi baru
  • Pipeline otomatis dengan Scikit-learn
  • Memastikan preprocessing yang persis sama setiap kali

Training Dataset

Data historis berkualitas untuk pembelajaran model

  • {{ stats.total_data if stats else '157' }} records data kredit historis yang dikurasi
  • 5 fitur kategorik: Riwayat, Lama Usaha, Pendapatan, Jaminan, Jumlah
  • Label: Terima ({{ stats.persentase_terima if stats else '59' }}%) / Tolak ({{ stats.persentase_tolak if stats else '41' }}%)
  • Dataset seimbang untuk learning optimal
  • Keputusan dari tim analis kredit profesional

5-Fold Cross-Validation

Validasi ketat untuk mencegah overfitting

  • Stratified K-Fold: Data dibagi 5 bagian equal
  • Training pada 4 fold (80%), testing pada 1 fold (20%)
  • Diulang 5× dengan kombinasi fold berbeda
  • Stratified: proporsi kelas Terima/Tolak seimbang per fold
  • Menghasilkan metrik yang reliable dan representatif

Prediction Pipeline

Alur dari input user hingga keputusan real-time

  • Step 1: User input 5 kriteria via web form
  • Step 2: OrdinalEncoder transform kategori ke numerik
  • Step 3: CategoricalNB hitung P(Terima|Input) & P(Tolak|Input)
  • Step 4: Output keputusan + probabilitas kedua kelas
  • Waktu: <100ms per prediksi (real-time)

Transparansi & Interpretability

Explainable AI untuk keputusan yang dipercaya

  • Kedua probabilitas (Terima & Tolak) ditampilkan eksplisit
  • Contoh: "Terima 87.3%, Tolak 12.7%" → high confidence
  • Contoh: "Terima 52%, Tolak 48%" → borderline, perlu review
  • Memungkinkan audit dan understanding reasoning
  • Sesuai prinsip Responsible AI

Keunggulan Naive Bayes

Mengapa algoritma ini perfect untuk kredit mikro

  • Sangat cepat: Training dan inference <100ms per prediksi
  • Excellent untuk kategorik: CategoricalNB dirancang untuk tipe data ini
  • Dataset kecil OK: Tidak membutuhkan ribuan data seperti deep learning
  • Probabilitas eksplisit: Interpretable dan explainable
  • Simple namun powerful: Trade-off yang balanced

Asumsi Independence

"Naif" tapi tetap bekerja baik dalam praktik

  • Naive Bayes mengasumsikan semua fitur independen satu sama lain
  • Contoh asumsi: Riwayat Kredit tidak berpengaruh pada Lama Usaha
  • Kenyataan: Fitur mungkin berkorelasi (pendapatan tinggi ↔ riwayat baik)
  • Meskipun "naif", algoritma tetap performa baik untuk klasifikasi
  • Trade-off: Simplicity & speed vs perfect accuracy

Real-World Application

Implementasi di lembaga keuangan mikro

  • Use case: Approval kredit mikro on-the-spot di kantor cabang
  • Scalability: Dapat memproses ratusan pengajuan per jam
  • Integration: Web interface yang user-friendly untuk petugas
  • Decision support: Rekomendasi sistem + judgment petugas
  • Production-ready: Model disimpan (.pkl) untuk loading instant

5 Fitur Klasifikasi

Faktor kunci dalam penilaian kredit mikro

  • Riwayat Kredit: Track record pembayaran (Buruk, Cukup, Baik)
  • Lama Usaha: Stabilitas bisnis (<1 tahun, 1-3 tahun, >3 tahun)
  • Pendapatan/Bulan: Kapasitas finansial (Rendah, Sedang, Tinggi)
  • Jaminan: Risk mitigation (Ada, Tidak Ada)
  • Jumlah Pinjaman: Nominal relatif (Kecil, Sedang, Besar)

Keragaman Kombinasi Fitur

Pola nuansa dalam keputusan kredit

  • Riwayat kurang baik + Jaminan kuat + Pendapatan tinggi → Terima
  • Riwayat baik + Pinjaman terlalu besar untuk pendapatan → Tolak
  • Model belajar nuansa, bukan hanya aturan sederhana
  • Kombinasi fitur lebih penting daripada fitur individual
  • Mencerminkan kompleksitas keputusan kredit di dunia nyata

Data Cleaning & Quality

Preprocessing ketat untuk data berkualitas

  • No missing values: Semua record complete dan valid
  • No duplicates: Setiap record unik dan independent
  • Standardized categories: Tidak ada typo atau inkonsistensi
  • Validated labels: Keputusan dari expert, bukan arbitrary
  • Ready for encoding: OrdinalEncoder dapat bekerja smooth

Exploratory Data Analysis

Insight dari distribusi dan korelasi fitur

  • Riwayat Kredit "Baik" → tingkat approval jauh lebih tinggi

  • Pendapatan tinggi → kompensasi riwayat kurang baik dalam beberapa kasus
  • Insight memvalidasi pilihan fitur dan meaningful pattern

  • Prinsip Responsible AI untuk keputusan finansial
